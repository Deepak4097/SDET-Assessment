# AutomationExercise UI & API Test Automation Framework

## Overview

This repository contains automated UI and API test suites for the AutomationExercise application, developed using **Playwright** and **TypeScript**.

The framework follows the **Page Object Model (POM)** design pattern with reusable components to promote maintainability, scalability, and code reuse.

- Playwright
- TypeScript
- Node.js
- Page Object Model (POM)

# Defect Identified

## Search Functionality Returns Results Based on Category Matching

### Summary

Searching for the keyword **"top"** returns products whose names do not contain the searched keyword.

### Steps to Reproduce

1. Navigate to the **Products** page.
2. Search for **"top"**.
3. Review the search results.

### Expected Result

If the search functionality is intended to search by **product name**, only products whose names contain the keyword **"top"** should be returned.

### Actual Result

Products are returned even when the keyword is absent from the product name. Examples include:

- Little Girls Mr. Panda Shirt
- Colour Blocked Shirt – Sky Blue

### Analysis

Investigation indicates that the application searches against **product categories (and potentially other metadata)** in addition to product names. Consequently, products belonging to categories such as **"Tops"** are included in the search results.

The API returns the same dataset as the UI, indicating that the frontend behavior is consistent with the backend implementation.

### Recommendation

Business clarification is required to confirm whether product search is intended to match:

- Product names only
- Product categories
- Tags
- Descriptions
- Other searchable metadata

# Blocked Test Case

## TC09 / TA09 – Verify Product Sorting by Price (Low → High)

### Status

**Blocked**

### Reason

The current version of AutomationExercise does not provide functionality to sort products by price through either the user interface or the available API.

Specifically, the application lacks:

- Sort dropdown
- Sort button
- Filter control
- API parameter supporting price sorting

As a result, the expected behavior cannot be executed or validated. Automation for this scenario can proceed only after the required sorting functionality is implemented.

# Known Limitations

AutomationExercise occasionally redirects to:

`https://automationexercise.com/#google_vignette`

This redirect is generated by an external Google advertisement and is outside the scope of the application under test. It may occasionally interrupt test execution and introduce intermittent failures.

The automation framework does not interact with or validate advertisement content.

---

# Flakiness Considerations

Intermittent failures may occur due to external factors, including:

- Redirection to the Google advertisement page (`#google_vignette`)
- Network latency
- Variable application response times
- Dynamic page loading delays

Where possible, the framework mitigates these issues through Playwright's built-in auto-waiting mechanisms and reusable synchronization utilities.
